What Mixed-Methods Studio does

Mixed-Methods Studio takes survey data that combines closed-ended items (Likert scales, ratings, demographics) with open-ended comments and walks you through coding, categorizing, integrating, and reporting on it. The work that traditionally lives across NVivo, SPSS, and Word collapses into one flow.

You bring a dataset. The studio writes themes from the open-ended responses, codes every row with a theme and a sentiment, builds derived variables you can analyze with the closed-ended items, drafts integration paragraphs that bring the quant and qual together, runs a methodological audit, and exports a draft report in DOCX or PDF.

You stay in control of every decision. The studio recommends; you accept or adjust. Themes can be renamed, merged, or recoded. Suggested analyses can be ignored. The narrative drafts can be rewritten.

The three phases

The studio runs in three phases. Each unlocks once the previous is complete. The studio jumps you to the next phase automatically, so you can follow the path or click any unlocked phase directly.

Phase 1. Get data in. Upload a file, link an existing ReliCheck survey, or paste rows. The studio expects one row per response.
Phase 2. Builder. Review the rows, run the theme builder, and confirm the per-response coding.
Phase 3. Analyze and report. Eight tabs of analysis, ending with an editable report you can export.

Phase 1: Get data in

Three intake routes. Pick whichever matches the data you have.

  • Upload a file. CSV, TSV, XLSX, XLS, SAV (SPSS), or DTA (Stata). The studio reads the headers and asks you which columns are which. One column needs to be an open-ended response. Optional: numeric scores, group labels (gender, school, condition), and question IDs if multiple open-ended items share rows.
  • Link a ReliCheck survey. If the survey already lives in your account, the studio reads it directly. Useful when you have an active distribution and want analysis as responses arrive.
  • Paste rows. For quick tests. Open-ended text in one column, optional score in another.

After upload, the studio shows a preview and a Quality Brief. The brief flags rows that look broken (empty, single-character, copy-paste duplicates) so you can clean before coding. Cleaning here saves time downstream.

The pathway matters. If your data has both Likert scores and open-ended comments per row, pick the scores + comments pathway. This unlocks the Score-to-Theme and Alignment tabs in Phase 3.

Phase 2: Builder

This is where the studio writes themes from your open-ended responses and codes every row.

  1. Review and clean. The Builder shows the rows the Quality Brief flagged. Remove, edit, or keep each. The studio also lets you remap which column is the open-ended text if the auto-pick was wrong.
  2. Run the build. The studio runs a two-pass process: it reads a discovery sample to surface candidate themes, then codes the remaining responses in batches. On 100 to 500 responses this takes 30 to 90 seconds. Larger datasets take longer; the studio shows progress.
  3. Review categories. You see a list of themes with example quotes. Rename anything that does not match how you would say it in a paper. Merge themes that overlap. Mark themes you want to drop. The category list is editable for the life of the project.

If the theme list looks off on your data, run the build again. The discovery sample is random, so a second run on the same dataset often surfaces a cleaner set. The studio remembers your edits across runs.

Phase 3: Analyze and report

Eight tabs of analysis. The studio recommends an order at the top of the tab row. The tabs are not locked: a doctoral researcher with a specific question can jump straight to the relevant tab. The recommended order, for first-time users:

1. Categories. Review the themes the Builder discovered. Rename, merge, or cluster before analysis.
2. Dataset. Sanity-check the per-response coding and the new variables the studio wrote into your dataset (theme, sentiment, derived columns).
3. Score-to-Theme (scores + comments pathway only). How each Likert score distributes across themes. A first read on whether the words agree with the numbers.
4. Alignment (scores + comments pathway only). Where the numbers and the words agree or diverge per group.
5. Analysis. The studio suggests IV/DV pairs and a recommended test. You pick the ones that match your question and run them. The math runs from a built-in stats library.
6. Joint Display. Quant and qual side-by-side per group. The mixed-methods table reviewers expect to see.
7. Integration. Drafted paragraphs that integrate the quant and qual findings. Edit them in the Report tab.
8. Strength Check. A methodological audit: sample sizes, theme coverage, missing pieces.
9. Report. Assemble and export the full mixed-methods report.

On the Analysis tab, the studio recommends a test (correlation, t-test, chi-square, ANOVA, regression) based on the variable types. The recommendation is a starting point, not a verdict. Read the test result with the lens you would apply to any quantitative output: effect size, sample size, assumption fit. The studio shows the assumption checks alongside the result.

On the Joint Display, you see the quant column (means, proportions) and the qual column (top themes, example quotes) per group. This is the table most mixed-methods journals expect to see in a paper.

On the Integration tab, the studio drafts one paragraph per theme that brings together the quant patterns and the qual evidence. These drafts are starting points. Edit them in the Report tab using the rich-text editor and the rewrite tool.

On Strength Check, the studio gives a per-project audit: sample size adequacy, theme coverage, alignment depth, missing pieces. Use it as a self-check before sharing your report.

Exporting your work

From the Report tab, you can export the assembled report as DOCX or PDF. The DOCX is editable. The PDF is a clean read-only version.

From the Dataset tab, you can export the coded dataset as CSV or as a SPSS-ready file (with value labels and variable labels preserved).

From the Categories tab, you can export the theme list with example quotes as CSV.

Your work is saved on every step. You can leave and come back. Exports are unlimited.

Citing the methods

If you publish work that used Mixed-Methods Studio, please describe the method in your own words. A starting language template:

Open-ended responses were coded using a software-assisted approach (ReliCheck Mixed-Methods Studio, closed beta). The system performed inductive theme discovery on a sample of responses, then coded the remaining responses against the resulting category set in batched passes. The author reviewed and revised the categories before analysis. Joint displays and integration paragraphs were generated by the system and revised by the author.

If a reviewer or committee member asks how the coding was done, you can describe the two-pass process: discovery sample to surface candidate themes, batched coding against the resulting set, author review at every step. The author retains full editorial control over theme names, category structure, integration narrative, and final report.

Sending feedback

The dark Feedback button sits in the bottom-right of every Studio page. Click it to open a short form: an optional rating, a comment box, and an auto-attached context line so we know which page or tab you were on. We read every note.

Useful things to send: a label that confused you, a result that looks wrong, a feature you wish existed, a step that took longer than you expected. Useful things to not send: requests for analyses outside the mixed-methods workflow (those belong in the standard project view).

If something goes wrong

The studio is in closed beta. Expect rough edges. If something breaks:

  • The page won't load or shows a spinner forever. Refresh the browser. Your work is saved server-side; you will not lose progress.
  • The Builder returns no themes or weird themes. Run it again. The discovery sample is random; a second run often produces a cleaner set on the same data.
  • An export fails. Try the other format (DOCX instead of PDF, or vice versa). The Report tab keeps your edits regardless of export status.
  • Anything else. Send a feedback note describing what you clicked and what you expected to see. The auto-attached context tells us where in the flow you were.
