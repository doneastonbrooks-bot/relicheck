ReliCheck App Restructure

One platform. Twelve interface levels. Guided for beginners, deep for experts, report-ready for teams. This document captures the architecture before any code is written, so the build can hold tightly to a single coherent shape.

1The governing principle: guided sophistication

Every user enters through the same simple workflow. The depth of analysis, explanation, recommendations, and reporting expands by membership level and user expertise. One app, progressive depth.

A new user should feel

"I know what to do next."

Clear paths, plain language, recommended next step always visible. The app does not ask the user to know what analysis they need; it asks what they have and what they want to understand.

An advanced user should feel

"The methodology is not hidden from me."

Full statistical tables, assumptions, item-level diagnostics, and exportable methodology one click below the plain-language layer. Credibility is always available.

4The two intake flows

Two clean paths from the home dashboard. Both end at the same analysis surface; the build steps differ.

Flow A

User creates an instrument in ReliCheck

  1. Start Project
  2. Choose: Create Survey or Create Test
  3. Select method: scratch, template, AI builder, or import existing questions
  4. Build the instrument
  5. Run Pre-Publish Readiness Check
  6. Revise weak questions
  7. Distribute or export the instrument
  8. Collect responses
  9. Analyze results
  10. Generate report

Flow B

User uploads existing survey or test data

  1. Start Project
  2. Choose: Upload Data
  3. Select data type: survey, test, dataset, or open-ended
  4. Upload file (CSV, Excel, Qualtrics, SurveyMonkey, Google Forms, SPSS later)
  5. Map columns to variable types (wizard-assisted)
  6. Confirm what each column means
  7. Run Data Quality Scan
  8. Choose analysis path (or accept recommendation)
  9. View dashboard
  10. Generate report

5The dual-mode dashboard

Not two separate dashboards. One dashboard with two readings of the same numbers. Switching between modes never changes the math; it changes the language.

Mode 1 . Guided View

For HR, leadership, business users, school admins, nontechnical evaluators

Plain-language summaries, strength scores, color-coded readiness indicators, "what this means" explanations, recommended next steps, AI-generated narrative summaries, simple charts, report-ready language.

ReliCheck found that your employee engagement scale is moderately reliable. Most items are working together, but two questions may be weakening the scale. Before using this survey for major decisions, consider revising Item 4 and Item 9.
Mode 2 . Technical View

For researchers, evaluators, advanced analysts, institutional researchers, psychometric users

Full statistical tables, assumptions, sample-size warnings, confidence intervals, effect sizes, reliability coefficients, factor loadings, KMO and Bartlett's test, correlation matrices, regression outputs, missing-data diagnostics, item-level statistics, exportable tables, methodology notes.

Cronbach's alpha = .82, McDonald's omega = .85, average inter-item correlation = .41. Item 4 produced the lowest corrected item-total correlation at .22. Removing Item 4 would increase alpha to .86.

6The summary-first UI pattern

Every analysis page follows the same five-step layered structure. The pattern repeats across reliability, factor analysis, descriptive, open-ended, group comparison, pre/post, predictors, and test-item dashboards.

01

What ReliCheck found

One-paragraph headline. Strength score or verdict pill.

02

Why it matters

Plain-language explanation for the audience that has to act on it.

03

What to do next

Specific item-level or analytic recommendations.

04

Technical details

Expandable panel with coefficients, tables, assumptions, diagnostics.

05

Add to report

Send this section to Executive, Technical, or Full Evidence report.

8Report levels (orthogonal to tier)

Think by report depth, not only by membership tier. Every analysis can produce reports at any of these levels, with depth-appropriate language and content.

Level Audience What it includes
1 . Basic Summary Beginners and lower tiers What was analyzed, key findings, simple charts, basic recommendations, plain-language interpretation. No heavy methods.
2 . Practical Recommendations Users who need to improve a survey, test, or dataset Strengths, weaknesses, items to revise, data-quality concerns, recommended next steps, suggested rewritten items. One of ReliCheck's biggest differentiators.
3 . Executive Report Leaders and teams High-level findings, decision implications, key risks, recommended actions, charts, plain-language evidence summary. Useful for HR, business, education, marketing, and program evaluation leaders.
4 . Technical Report Advanced users, methodologists Methods, assumptions, statistical tables, reliability coefficients, factor results, regression outputs, limitations, interpretation cautions, appendix. The credibility layer.
5 . Full Evidence Report High-tier users; submission, defense, accreditation Executive summary, data quality review, all analysis sections, technical appendix, instrument improvement recommendations, AI narrative, charts, export tables, methodology notes, limitations, action plan. The premium report.

For Team, Business, and Enterprise tiers. Six roles, role-based access controls, project-level scoping. Crucial for HR, education, program evaluation, and research clients.

10Privacy-centered design

Privacy is built into the app, not added at the end. Especially important for HR (employee data), education (student data, FERPA), and research (IRB compliance) use cases.

P1

Small subgroup suppression

Hide subgroups below the k-anonymity threshold entirely, not just masked.

P2

Role-based access

Restrict raw response access by role; surface only what each role needs.

P3

Report redaction

Author-controlled hide/show of sensitive segments before share.

P4

Anonymous response mode

Strip identifiers at upload or collection time.

P5

Data deletion controls

User-initiated deletion with admin audit trail.

P6

Export permissions

Restrict who can export raw data versus aggregate reports.

P7

Audit trail (Enterprise)

Who saw what, when, and what they exported.

11AI integration principles

AI is embedded in the workflow, not a separate gimmick. The standard: AI can explain and draft. ReliCheck analysis must anchor the claim.

Stage Where AI shows up
Instrument creationImprove a question, detect bias, rewrite at a clearer reading level, align an item to a construct, suggest response options, generate a survey from a goal.
Upload and mappingIdentify variable types, detect likely scales, find open-ended fields, suggest grouping variables, flag messy columns.
AnalysisExplain a result, answer "what does this mean?", flag interpretation cautions, summarize a chart, compare groups in plain language.
ReportsWrite executive summary, technical methods section, limitations, recommendations, board-ready language, client-ready language.

12Look, feel, and voice

Calm, credible, structured. Closer to a clean analytics workspace plus a survey-improvement lab plus a report-writing assistant. Not SurveyMonkey.

Visual style

Calm, professional, evidence-focused

Light background. Deep navy or charcoal text. Coral/orange accent for key actions. Soft cards. Clear section headings. Plenty of spacing. Tables only when needed. Expandable technical panels. Report-ready chart previews.

Tone

Plain-language first, technical when requested

Evidence-focused. Direct about limitations. Helpful without sounding gimmicky. Avoid academic-only voice. Avoid playful or cold voices. Avoid the "wall of statistics" problem on first paint.

Anti-patterns to avoid

Six design mistakes that would undo guided sophistication.

  • Statistical-first dashboards: never let the first paint be a wall of coefficients. Advanced statistics should be available, never forced.
  • Forcing analysis choice upfront: do not make the user pick "Cronbach's alpha vs KMO vs IRT" before any data is in. Let ReliCheck recommend.
  • Hiding the technical layer: do not collapse methodology completely. Advanced users need it one click away to trust the platform.
  • Reports as an afterthought: reports are a main outcome, not a leftover feature. Every analysis page sends to a report.
  • AI as a separate page: AI lives inside the workflow. A standalone "AI" page is for marketing, not for the app.
  • Different apps per tier: one product, deeper access by plan. Never make Free and Enterprise feel like different products.
